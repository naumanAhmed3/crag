# BM25 — the lexical baseline you should never skip

BM25 is the 30-year-old relevance function that powers Lucene,
Elasticsearch, and the sparse leg of nearly every serious hybrid search
system. The intuition: a query term contributes more to a document's
score when it appears frequently in that document (term frequency,
saturating), but less when it appears in many documents across the
corpus (inverse document frequency, dampening common words). A length
normalisation factor punishes overlong documents that "win" by sheer
word count.

## The formula in one line

For a document `d`, a query `q` containing terms `t`:

```
BM25(d, q) = Σ_t IDF(t) · (tf(t,d) · (k1 + 1)) / (tf(t,d) + k1 · (1 − b + b · |d|/avgdl))
```

`k1` (1.2 to 2.0) controls term-frequency saturation; `b` (0 to 1,
usually 0.75) controls length normalisation. The defaults in
`rank_bm25` are sane for technical text.

## Why hybrid retrieval needs BM25

Dense retrievers are great at semantic similarity but blind to literal
tokens. Three queries where BM25 systematically wins:

- Identifiers, version numbers, error codes: `"CVE-2024-3094"`,
  `"err_code=4017"`. Embedding models map these to noise.
- Rare proper nouns: a person's name that never appeared in training.
- Highly specific terminology where the user is quoting back something
  they read: `"hnsw.ef_search parameter"`.

Conversely, BM25 misses paraphrase. The fusion of the two is not a
nice-to-have — it routinely adds 5 to 15 points of Recall@5 over
either alone on technical corpora.

## Where BM25 starts to hurt

Three failure modes worth knowing:

1. **Synonym deafness**: a document that says "automobile" gets no
   credit when the query says "car." Stemming + lemmatisation help a
   little; cross-lingual makes BM25 essentially unusable on its own.
2. **Long passage penalty**: very long chunks suffer from the length
   normaliser. Either chunk smaller or set `b` low (0.25).
3. **Tokenisation hazard**: the same `rank_bm25` index that uses
   word-boundary tokenisation will mis-tokenise code: `"foo.bar()"`
   becomes one token `foobar`. For code-heavy corpora, swap to a
   regex-based tokeniser that splits on non-alphanumeric runs.
